Detailed analysis methods and calculation algorithms

Cluster Buy Analysis

10minutes window

Purpose: Identifies tokens with coordinated multiple buyer activity

Methodology:

  • Groups transactions by token_mint in 10-minute windows
  • Counts unique wallet addresses (buyers)
  • Filters tokens with 3+ unique buyers
  • Sorts by total transaction count

SQL Query:

SELECT token_mint, COUNT(*) as tx_count, COUNT(DISTINCT wallet) as unique_buyers FROM events WHERE side = 'BUY' AND ts >= NOW() - INTERVAL '10 minutes' GROUP BY token_mint HAVING COUNT(DISTINCT wallet) >= 3 ORDER BY tx_count DESC
Signal threshold: ≥3 unique buyers

Whale Moves Analysis

30minutes window

Purpose: Tracks large-volume purchases indicating whale activity

Methodology:

  • Filters buy transactions by SOL spent threshold
  • Monitors 30-minute time window
  • Includes trader identification from walletMap
  • Orders by volume and recency

SQL Query:

SELECT token_mint, wallet, sol_spent, ts FROM events WHERE side = 'BUY' AND sol_spent > 50 AND ts >= NOW() - INTERVAL '30 minutes' ORDER BY sol_spent DESC, ts DESC
Volume threshold: ≥50 SOL per transaction

Volume Surge Analysis

15minutes window

Purpose: Detects sudden spikes in trading volume

Methodology:

  • Sums SOL volume per token over 15 minutes
  • Aggregates total transaction count
  • Filters tokens above volume threshold
  • Prioritizes highest volume increases

SQL Query:

SELECT token_mint, SUM(sol_spent) as total_volume, COUNT(*) as tx_count FROM events WHERE side = 'BUY' AND ts >= NOW() - INTERVAL '15 minutes' GROUP BY token_mint HAVING SUM(sol_spent) > 100 ORDER BY total_volume DESC
Volume threshold: ≥100 SOL total volume
